Tic-Tac-Toe

Layout

Vue things

  • vuex: has 4 props; [state, mutations, getters, actions]
    • state: similar to vue’s data object
    • mutations: mutates states synchronously, similar to events
    • actions: triggers mutations asynchronously with commit()
    • getters: similar to computed prop for vuex stroes.
  • vuex store: Plays a role of centralized shared storage for all Vue components.
  • mapState: a helper to connect computed prop to multiple states, and make it easier to load states with shorter codes. Other vuex props have their own helpers.
  • this.$store.commit: actually changes states by committing mutations.
  • <slot />: Make children compontnents overridable and expandable in their parents. Deprecated in vue3 with new v-slot.

How to play

  • There are 9 cells in a table.
  • Make a line of your symbols in any row, in any column, or in one of both diagonals.

Game algorithm

It’s Tic-Tac-Toe!

  • Mainly focused on how parent-children components handle data/state.
  • But Vuex can handle those roles instead. Components has no need to communicate themselves through emit events.

Reaction Time Tester

Layout

Vue things

  • <template>: Child Template can group other elements.
  • v-show: Conditionally ‘displays’ the element by changes of its CSS.
  • v-bind: Class binding for the app to change style (color).
  • computed: Similar to method, but it only runs when injected element changes

How to play

  • Click the box to go ready - steady.
  • Click ASAP again when the box is turned into green.

Game algorithm

This is simple.

  • The app has an array of times of the user’s reaction results
  • And has a state to change colors bound to the div class.
  • From color of ‘steady’ to color of ‘go’ in random time
  • Through div_id.class selector, we can select colors conditionally.
  • To get average time, Array.reduce() is used.
  • Reset button resets the app to the initial state.

Number Cracker

Link to the Game

Number Cracker

Layout

Vue things

  • v-on:submit.prevent in form element to block refreshign the page. Then button ‘submit’ will do another thing.
  • v-for will show list-items augmented as the user clicks the submit button.
  • v-model will get a value from input box and send it to ‘value’ in data.

How to play

  • At start, 4-digit unique random number is selected.
  • The number is ranged between 1 - 9.
  • The user can guess 10 times.
  • If a number is in the list but in another position, it’s counted as ball.
  • If a number is in a right place, it’s counted as strike.
  • If the user get the correct number within 10 tries, the user wins.

Game algorithm

There is a helper function to initially pick 4 random unique numbers.

  • There is an candidates array of 9 numbers (1 - 9).
  • To implement random unique number, used Math.floor() and Math.random().
  • To make a chosen number unique, used candidates.splice(RANDOM_CHOSEN_NUMBER)
  • Pick a number 4 times in a loop, in each loop push the random number to a new array.

Main game algorithm is as follows.

  • Has 4 data(or states) elements
  • answer: get array of random number
  • trials: an array of user trials
  • value: get user’s trial numbers from input box
  • reuslt: a string to display a title or game progress
  • There is one method for the game lifecyle.
    • First, compare input with the answer. array.join(‘’) is used to make the array into 4 digit number to match with input numbers.
    • If the input is wrong, analyze the input numbers (compare with answer).
    • Displays the result in the list.
    • Repeat till get the answer or waste all 10 chances.
